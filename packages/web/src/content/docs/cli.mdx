The Whykido CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
whykido
```

But it also accepts commands as documented on this page. This allows you to interact with Whykido programmatically.

```bash
whykido run "Explain how closures work in JavaScript"
```

---

### tui

Start the Whykido terminal user interface.

```bash
whykido [project]
```

#### Flags

| Flag         | Short | Description                                |
| ------------ | ----- | ------------------------------------------ |
| `--continue` | `-c`  | Continue the last session                  |
| `--session`  | `-s`  | Session ID to continue                     |
| `--prompt`   |       | Prompt to use                              |
| `--model`    | `-m`  | Model to use in the form of provider/model |
| `--agent`    |       | Agent to use                               |
| `--port`     |       | Port to listen on                          |
| `--hostname` |       | Hostname to listen on                      |

## Commands

The Whykido CLI also has the following commands.

---

### agent

Manage agents for Whykido.

```bash
whykido agent [command]
```

---

### attach

Attach a terminal to an already running Whykido backend server started via `serve` or `web` commands.

```bash
whykido attach [url]
```

This allows using the TUI with a remote Whykido backend. For example:

```bash
# Start the backend server for web/mobile access
whykido web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
whykido attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
whykido agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
whykido agent list
```

---

### auth

Command to manage credentials and login for providers.

```bash
whykido auth [command]
```

---

#### login

Whykido is powered by the provider list at [Models.dev](https://models.dev), so you can use `whykido auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/whykido/auth.json`.

```bash
whykido auth login
```

When Whykido starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
whykido auth list
```

Or the short version.

```bash
whykido auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
whykido auth logout
```

---

### github

Manage the GitHub agent for repository automation.

```bash
whykido github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
whykido github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
whykido github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
whykido mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
whykido mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
whykido mcp list
```

Or use the short version.

```bash
whykido mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
whykido mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
whykido mcp auth list
```

Or use the short version.

```bash
whykido mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
whykido mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
whykido mcp debug <name>
```

### models

List all available models from configured providers.

```bash
whykido models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
whykido models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in Whykido.

```bash
whykido models --refresh
```

### run

Run whykido in non-interactive mode by passing a prompt directly.

```bash
whykido run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "whykido run"
whykido run Explain the use of context in Go
```

You can also attach to a running `whykido serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
whykido serve

# In another terminal, run commands that attach to it
whykido run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag         | Short | Description                                                        |
| ------------ | ----- | ------------------------------------------------------------------ |
| `--command`  |       | The command to run, use message for args                           |
| `--continue` | `-c`  | Continue the last session                                          |
| `--session`  | `-s`  | Session ID to continue                                             |
| `--share`    |       | Share the session                                                  |
| `--model`    | `-m`  | Model to use in the form of provider/model                         |
| `--agent`    |       | Agent to use                                                       |
| `--file`     | `-f`  | File(s) to attach to message                                       |
| `--format`   |       | Format: default (formatted) or json (raw JSON events)              |
| `--title`    |       | Title for the session (uses truncated prompt if no value provided) |
| `--attach`   |       | Attach to a running whykido server (e.g., http://localhost:4096)  |
| `--port`     |       | Port for the local server (defaults to random port)                |

### serve

Start a headless Whykido server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
whykido serve
```

This starts an HTTP server that provides API access to whykido functionality without the TUI interface. Set `WHYKIDO_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `whykido`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

```bash
whykido session [command]
```

---

#### list

List all Whykido sessions.

```bash
whykido session list
```

##### Flags

| Flag          | Short | Description                          |
| ------------- | ----- | ------------------------------------ |
| `--max-count` | `-n`  | Limit to N most recent sessions      |
| `--format`    |       | Output format: table or json (table) |

---

### stats

Show token usage and cost statistics for your Whykido sessions.

```bash
whykido stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
whykido export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or Whykido share URL.

```bash
whykido import <file>
```

You can import from a local file or an Whykido share URL.

```bash
whykido import session.json
whykido import https://opncd.ai/s/abc123
```

---

### web

Start a headless Whykido server with a web interface.

```bash
whykido web
```

This starts an HTTP server and opens a web browser to access Whykido through a web interface. Set `WHYKIDO_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `whykido`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
whykido acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall Whykido and remove all related files.

```bash
whykido uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates whykido to the latest version or a specific version.

```bash
whykido upgrade [target]
```

To upgrade to the latest version.

```bash
whykido upgrade
```

To upgrade to a specific version.

```bash
whykido upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

Whykido can be configured using environment variables.

| Variable                              | Type    | Description                                       |
| ------------------------------------- | ------- | ------------------------------------------------- |
| `WHYKIDO_AUTO_SHARE`                 | boolean | Automatically share sessions                      |
| `WHYKIDO_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows            |
| `WHYKIDO_CONFIG`                     | string  | Path to config file                               |
| `WHYKIDO_CONFIG_DIR`                 | string  | Path to config directory                          |
| `WHYKIDO_CONFIG_CONTENT`             | string  | Inline json config content                        |
| `WHYKIDO_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                   |
| `WHYKIDO_DISABLE_PRUNE`              | boolean | Disable pruning of old data                       |
| `WHYKIDO_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates          |
| `WHYKIDO_PERMISSION`                 | string  | Inlined json permissions config                   |
| `WHYKIDO_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                           |
| `WHYKIDO_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads            |
| `WHYKIDO_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                        |
| `WHYKIDO_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction              |
| `WHYKIDO_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)  |
| `WHYKIDO_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`             |
| `WHYKIDO_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                  |
| `WHYKIDO_CLIENT`                     | string  | Client identifier (defaults to `cli`)             |
| `WHYKIDO_ENABLE_EXA`                 | boolean | Enable Exa web search tools                       |
| `WHYKIDO_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`               |
| `WHYKIDO_SERVER_USERNAME`            | string  | Override basic auth username (default `whykido`) |

---

| `WHYKIDO_EXPERIMENTAL_ICON_DISCOVERY`          | boolean | Enable icon discovery                   |
| `WHYKIDO_EXPERIMENTAL_DISABLE_COPY_ON_SELECT`  | boolean | Disable copy on select in TUI           |

| `WHYKIDO_EXPERIMENTAL_BASH_DEFAULT_TIMEOUT_MS` | number  | Default timeout for bash commands in ms |
| `WHYKIDO_EXPERIMENTAL_OUTPUT_TOKEN_MAX`        | number  | Max output tokens for LLM responses     |
| `WHYKIDO_EXPERIMENTAL_FILEWATCHER`             | boolean | Enable file watcher for entire dir      |
| `WHYKIDO_EXPERIMENTAL_OXFMT`                   | boolean | Enable oxfmt formatter                  |
| `WHYKIDO_EXPERIMENTAL_LSP_TOOL`                | boolean | Enable experimental LSP tool            |
