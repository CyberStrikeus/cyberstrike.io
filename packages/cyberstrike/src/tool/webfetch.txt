- Fetches content from a specified URL with full HTTP method support
- Supports GET, POST, PUT, DELETE, and PATCH methods
- Supports custom request body (string or JSON object)
- Supports custom HTTP headers
- Returns content in requested format (markdown, text, html, or json)

Usage notes:
  - IMPORTANT: if another tool is present that offers better web fetching capabilities, is more targeted to the task, or has fewer restrictions, prefer using that tool instead of this one.
  - The URL must be a fully-formed valid URL
  - HTTP URLs will be automatically upgraded to HTTPS
  - Method options: "GET" (default), "POST", "PUT", "DELETE", "PATCH"
  - Format options: "markdown" (default), "text", "html", or "json"
  - Body: For POST/PUT/PATCH, objects are automatically serialized to JSON
  - Headers: Custom headers can override default User-Agent, Accept, etc.
  - Results may be summarized if the content is very large
