The Web Application agent is a specialized security testing agent equipped with comprehensive knowledge of the OWASP Web Security Testing Guide (WSTG) v4.2, featuring 109 documented tests across 12 categories.

## Overview

The Web Application agent combines deep security knowledge with automated tooling to perform thorough web application penetration testing. It's pre-loaded with specialized skills for SQL injection testing and OWASP WSTG methodology.

### Agent Configuration

| Property | Value |
|----------|-------|
| **Agent ID** | `web-application` |
| **Color** | Red |
| **Pre-loaded Skills** | `owasp-wstg`, `sql-injection` |
| **Allowed Tools** | Browser, Bash, WebFetch, WebSearch, Read, Glob, Grep |

### Launch the Agent

```bash
# Start Cyberstrike with web application agent
cyberstrike --agent web-application

# Or switch during session
/agent web-application
```

## Knowledge Base

The Web Application agent has access to extensive security knowledge through the skills system.

### OWASP WSTG v4.2 Skill

The agent is pre-loaded with the complete OWASP Web Security Testing Guide v4.2, providing systematic testing methodology for 109 security tests across 12 categories.

### SQL Injection Skill

Comprehensive SQL injection testing knowledge including:

- Detection payloads (basic, error-based)
- Database fingerprinting queries (MySQL, PostgreSQL, MSSQL, Oracle, SQLite)
- Union-based injection methodology
- Blind SQL injection (boolean-based, time-based)
- Out-of-band (OOB) techniques with DNS exfiltration
- WAF bypass techniques
- SQLMap command reference

## Testing Methodology

The agent follows a structured approach:

```
┌─────────────────────────────────────────────────────────────┐
│                    TESTING METHODOLOGY                       │
├─────────────────────────────────────────────────────────────┤
│  1. RECONNAISSANCE                                           │
│     └── Information gathering, fingerprinting               │
│                                                              │
│  2. MAPPING                                                  │
│     └── Application structure, entry points                 │
│                                                              │
│  3. DISCOVERY                                                │
│     └── Vulnerability identification per WSTG category      │
│                                                              │
│  4. EXPLOITATION                                             │
│     └── Proof of concept, impact demonstration              │
│                                                              │
│  5. REPORTING                                                │
│     └── Findings with CVSS scores, remediation              │
└─────────────────────────────────────────────────────────────┘
```

| Severity | CVSS Score | Example Vulnerabilities |
|----------|------------|------------------------|

## Best Practices

1. **Scope Definition** - Clearly define testing boundaries and excluded endpoints
2. **Authentication** - Provide valid credentials for comprehensive authenticated testing
3. **Rate Limiting** - Configure appropriate delays to avoid triggering WAF/rate limits
4. **Evidence Collection** - Use HAR export for complete traffic capture
5. **Verification** - Manually verify critical and high severity findings

<Aside variant="caution">
  Always obtain proper written authorization before testing web applications. Unauthorized security testing is illegal and unethical.
</Aside>
