The Bug Hunter agent is a specialized security testing agent equipped with comprehensive knowledge of reconnaissance methodology, attack surface discovery, and bug bounty hunting techniques.

## Overview

The Bug Hunter agent combines deep reconnaissance knowledge with automated tooling to maximize attack surface discovery for bug bounty programs. It's pre-loaded with specialized skills for subdomain enumeration, content discovery, and vulnerability pattern detection.

### Agent Configuration

| Property | Value |
|----------|-------|
| **Agent ID** | `bug-hunter` |
| **Color** | Green |
| **Pre-loaded Skills** | `recon-methodology`, `owasp-wstg`, `subdomain-takeover`, `nuclei-templates` |
| **Allowed Tools** | Browser, Bash, WebFetch, WebSearch, Read, Glob, Grep, Memory |

### Launch the Agent

```bash
# Start Cyberstrike with bug hunter agent
cyberstrike --agent bug-hunter

# Or switch during session
/agent bug-hunter
```

## Report Template

```markdown
## Summary
[One sentence describing the vulnerability]

## Affected Asset
- **URL:** https://target.com/vulnerable-endpoint
- **Parameter:** redirect_url
- **Method:** GET

## Vulnerability Type
Open Redirect (CWE-601)

## Severity
Medium (CVSS 5.4)

## Steps to Reproduce
1. Navigate to https://target.com/login
2. Append ?redirect_url=//evil.com to the URL
3. Complete login process
4. Observe redirect to evil.com

## Proof of Concept
[Screenshot or video]

## Impact
An attacker can craft a malicious link that appears to be from target.com
but redirects users to a phishing page. This can be used for:
- Credential harvesting
- Malware distribution
- OAuth token theft

## Remediation
1. Validate redirect URLs against a whitelist
2. Use relative URLs only
3. Implement destination verification

## References
- https://cwe.mitre.org/data/definitions/601.html
- https://owasp.org/www-project-web-security-testing-guide/
```

## Best Practices

1. **Read Program Policy** - Understand scope, rules, and exclusions
2. **Start Passive** - Begin with non-invasive reconnaissance
3. **Track Progress** - Use Memory tool to avoid duplicate work
4. **Verify Impact** - Demonstrate real security impact
5. **Write Clear Reports** - Help triagers reproduce issues
6. **Focus on Unique Issues** - Automated findings are often duplicates

<Aside variant="caution">
  Always respect program scope and rate limits. Aggressive scanning can result in program bans and legal issues.
</Aside>
