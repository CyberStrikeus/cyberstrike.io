## System Requirements

Before installing Cyberstrike, verify your system meets these requirements:

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| Operating System | macOS 12+, Ubuntu 20.04+, Windows 10+ | macOS 14+, Ubuntu 22.04+ |
| RAM | 4 GB | 8 GB |
| Disk Space | 500 MB | 2 GB |
| Node.js | 18.0+ | 20.0+ |
| Bun | 1.0+ | 1.1+ |

## Installing Cyberstrike

Choose your preferred installation method based on your operating system and package manager.

### npm (Recommended)

1. Open your terminal
2. Run the installation command:

```bash
npm install -g @cyberstrike-io/cli
```

3. Verify the installation:

```bash
cyberstrike --version
```

<div className="rounded-lg overflow-hidden my-6 border border-gray-200 dark:border-gray-700">
  <img src="/docs/images/screenshots/s01-installation.png" alt="Cyberstrike TUI after installation" className="w-full" />
</div>

### Bun

1. Install Bun if not already installed:

```bash
curl -fsSL https://bun.sh/install | bash
```

2. Install Cyberstrike with the `--trust` flag (required for postinstall):

```bash
bun install -g @cyberstrike-io/cli --trust
```

### Homebrew (macOS)

1. Add the Cyberstrike tap:

```bash
brew tap CyberStrikeus/tap
```

2. Install Cyberstrike:

```bash
brew install cyberstrike
```

```bash

```

## Authenticating with an AI Provider

<div className="rounded-lg overflow-hidden my-6 border border-gray-200 dark:border-gray-700">
  <img src="/docs/images/gifs/g02-auth-login.gif" alt="Cyberstrike auth login flow" className="w-full" />
</div>

Cyberstrike requires an AI provider to function. The following providers are supported:

| Provider | Free Tier | API Key Required |
|----------|-----------|------------------|
| OpenRouter | Yes | Yes |
| Anthropic | No | Yes |
| OpenAI | No | Yes |
| Google Gemini | Yes | Yes |
| Ollama | Yes (Local) | No |

### Setting Up Authentication

<div className="rounded-lg overflow-hidden my-6 border border-gray-200 dark:border-gray-700">
  <img src="/docs/images/screenshots/s02-auth.jpg" alt="Cyberstrike model selection dialog" className="w-full" />
</div>

1. Run the authentication command:

```bash
cyberstrike auth login
```

2. Select your provider from the list:

```
? Select provider
❯ OpenRouter (free tier available)
  Anthropic (API key from console.anthropic.com)
  OpenAI (ChatGPT Plus/Pro or API key)
  Google (API key from ai.google.dev)
  Ollama (local models)
  Other
```

3. Enter your API key when prompted

4. Verify your credentials:

```bash
cyberstrike auth list
```

### Using Environment Variables

Set API keys as environment variables instead of storing them:

```bash
# Anthropic
export ANTHROPIC_API_KEY="sk-ant-..."

# OpenAI
export OPENAI_API_KEY="sk-..."

# Google
export GOOGLE_API_KEY="AI..."

# OpenRouter
export OPENROUTER_API_KEY="sk-or-..."
```

Add these to your shell profile (`~/.bashrc`, `~/.zshrc`) for persistence.

## Running Your First Scan

### Starting Interactive Mode

Launch Cyberstrike in your target directory:

```bash
cd /path/to/target
cyberstrike
```

The terminal user interface (TUI) appears with:
- Input prompt at the bottom
- Response area in the center
- Status bar showing model and token usage

<div className="rounded-lg overflow-hidden my-6 border border-gray-200 dark:border-gray-700">
  <img src="/docs/images/screenshots/s03-tui-main.jpg" alt="Cyberstrike TUI main interface" className="w-full" />
</div>

### Executing a Quick Scan

Run a single command without entering interactive mode:

```bash
cyberstrike run "perform reconnaissance on example.com"
```

### Selecting a Security Agent

Cyberstrike includes specialized agents for different assessment types:

```bash
# Web application security
cyberstrike --agent web-application

# Cloud infrastructure
cyberstrike --agent cloud-security

# Network penetration testing
cyberstrike --agent internal-network

# Bug bounty reconnaissance
cyberstrike --agent bug-hunter
```

## Basic Configuration

Create a `cyberstrike.json` file in your project directory to set default options:

```json title="cyberstrike.json"
{
  "$schema": "https://cyberstrike.io/config.json",
  "model": "anthropic/claude-sonnet-4-20250514",
  "default_agent": "web-application"
}
```

When you run `cyberstrike` in this directory, it automatically loads these settings. The example above:

- Uses Claude Sonnet as the default AI model
- Starts with the web-application security agent

This eliminates the need to specify `--model` and `--agent` flags every time.

### Configuration Hierarchy

Cyberstrike loads configuration from multiple sources in this order:

1. Command line arguments (highest priority)
2. Environment variables
3. Project config (`cyberstrike.json`)
4. User config (`~/.cyberstrike/config.json`)
5. Default values (lowest priority)

For detailed configuration options, see the [Configuration Reference](/docs/configuration).
