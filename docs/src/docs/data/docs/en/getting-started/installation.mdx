Cyberstrike can be installed through multiple package managers. Choose the method that best fits your development environment.

## npm (Recommended)

The recommended installation method for most users.

```bash
npm install -g @cyberstrike-io/cli
```

Verify the installation:

```bash
cyberstrike --version
```

## Bun

For users preferring the Bun runtime:

```bash
# Install Bun if not already installed
curl -fsSL https://bun.sh/install | bash

# Install Cyberstrike with --trust flag
bun install -g @cyberstrike-io/cli --trust
```

<Aside variant="tip">
  The `--trust` flag is required because Bun blocks postinstall scripts by default. This allows the binary download to complete during installation.
</Aside>

## Homebrew (macOS)

For macOS users with Homebrew:

```bash
brew install CyberStrikeus/tap/cyberstrike
```

Or add the tap separately:

```bash
brew tap CyberStrikeus/tap
brew install cyberstrike
```

## curl (Linux/macOS)

Direct installation via shell script:

```bash
curl -fsSL https://cyberstrike.io/install.sh | bash
```

<Aside variant="caution">
  Always review installation scripts before executing them. The script source is available at [github.com/CyberStrikeus/cyberstrike.io](https://github.com/CyberStrikeus/cyberstrike.io).
</Aside>

## Docker

<Aside variant="caution">

## Verifying Installation

After installation, verify Cyberstrike is working:

```bash
# Check version
cyberstrike --version

# List available commands
cyberstrike --help
```

You should see the Cyberstrike banner and version number, followed by a list of available commands.

## Updating Cyberstrike

Update to the latest version:

```bash
# npm
npm update -g @cyberstrike-io/cli

# Bun
bun update -g @cyberstrike-io/cli

# Homebrew
brew upgrade cyberstrike

# Self-update (recommended)
cyberstrike upgrade
```

## Uninstalling

Remove Cyberstrike from your system:

```bash
# npm
npm uninstall -g @cyberstrike-io/cli

# Bun
bun remove -g @cyberstrike-io/cli

# Homebrew
brew uninstall cyberstrike
```

Configuration files are stored in `~/.cyberstrike/` and can be removed manually if needed.

## Troubleshooting

### Command Not Found

If `cyberstrike` is not found after installation:

1. Check your PATH includes npm global bin directory
2. Restart your terminal
3. Run `npm bin -g` to find the global bin path

### Permission Errors

On Linux/macOS, if you encounter permission errors:

```bash
# Fix npm permissions
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### Version Conflicts

If you have multiple Node versions:

```bash
# Use nvm to manage versions
nvm use 20
npm install -g @cyberstrike-io/cli
```

### Bun Postinstall Blocked

If Bun shows "Blocked 1 postinstall" and Cyberstrike says the binary is not found, reinstall with the `--trust` flag:

```bash
bun install -g @cyberstrike-io/cli --trust
```
